# sports-analytics-innovation
# Project Overview
**The "Power-Surge" Predictor:** This is a predictive analytics tool designed to forecast a player's season-level Home Run (HR) production using early-season counting statistics.

# Decision-Making Problem
This tool supports the **General Manager (GM)** during the Trade Deadline. It helps identify "undervalued" power hitters—players whose current HR totals are low, but whose secondary stats (like Walks and Doubles) suggest a massive power breakout is coming.

# Proposed Analytics Approach
We utilize a **Multiple Linear Regression** model. The approach uses readily available season-level statistics including Walks (BB), Doubles (2B), Triples (3B), Stolen Bases (SB), and Player Age. 

# Use by Decision Makers
A GM or Head of Scouting will interact with a "Target List" generated by this tool. The list ranks potential trade targets by their "Expected HR" vs. "Actual HR," allowing the front office to bid on players before their market value skyrockets.

## Prototype Enhancement
**Change:** Moving from a Baseline Linear Regression to a **Neural Network (MLP Regressor)** with hyperparameter tuning.

**Why this improves decision-making:** The original linear model assumed that a player's power grows at a constant rate. However, baseball aging is non-linear—players peak in their late 20s and decline sharply. By using a Neural Network, we improved our predictive accuracy (R²) from 0.44 to 0.46. This higher precision gives the GM more confidence when committing millions of dollars to a trade.

## Prototype Evaluation
**Decision:** The Neural Network enhancement **should** be integrated into the main project. 

**Influencing Factors:** Before final adoption, we need feedback from the **Scouting Department**. If the "black box" nature of the Neural Network makes it too difficult for scouts to explain *why* a player is ranked highly to the GM, we may need to stick with the simpler Linear Model to maintain organizational "buy-in."

## Reflection on Innovation and Version Control
- **Low-Risk Experimentation:** By using the `prototype` branch, I was able to build and tune a complex Neural Network without "breaking" the existing Linear Model that the GM was already using. This prevents technical errors from disrupting daily front-office operations.
- **Gaining Traction:** Version control allows us to show the "Evolution of an Idea." When decision-makers see the iteration history, it demonstrates that the analytics team is constantly refining the tool based on performance data, which builds institutional trust.
- **Alignment with Chapter 7:** This workflow mirrors the **Build Phase**. We moved from an abstract idea (Creative) to a tested version (Prototyping) and used "Engagement" (evaluating scout feedback) to decide on the final "Build" for the organization.
